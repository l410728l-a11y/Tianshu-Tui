## 1. 推广定位与受众画像

为了在短期内实现快速传播，我们将受众分为三类，并制定差异化的叙事方式：

1. **核心极客 & 终端重度用户 (The Hacker)**
   - *痛点*：受够了繁重的 IDE 插件，喜欢在 Vim/Tmux 极简终端中工作，同时极其在意 API 调用的账单成本。
   - *天枢叙事*：**“DeepSeek V4 稳态 95–99% 缓存命中率，把长会话账单显著压低”**。
2. **AI Agent 研究者 & 底层开发者 (The Agent Researcher)**
   - *痛点*：认为现在的 AI 编程工具只是简单的 `System Prompt + RAG + Tool Call`，没有解决 LLM 在运行时随着时间衰减而“越聊越笨”的服从性问题。
   - *天枢叙事*：**“端侧认知虚拟机 (CVM)，在不修改权重的前提下纠正 LLM 运行时缺陷”**。
3. **开源项目维护者 & 团队决策者 (The Contributor)**
   - *痛点*：担心 AI 在复杂项目中胡乱修改代码，引发测试崩溃或 Git 冲突。
   - *天枢叙事*：**“自带交付门禁与多会话并发协调锁的工业级智能体”**。

### 3.2 平台 B：V2EX / 开发者论坛技术贴（中文）

**帖子标题**：
> [开源发布] 天枢 (Tianshu) - 专为 DeepSeek 优化、稳态 95–99% 缓存命中率的终端 AI 编程智能体（附认知虚拟机 CVM 设计）

**帖子内容**：
> 各位 V 友，我们近期开源了一个完全自研的终端 AI 编程助手——**天枢 (Tianshu-Tui)**。
> 
> 传统的 AI 插件大多把上下文当成“黑盒”，频繁的上下文重写不仅让 API 账单暴涨，也让 AI 在多轮对话后因窗口溢出而“变笨”。我们做天枢的核心目标，就是把它打造成一个**能自感知、省钱、且有独立防错逻辑的智能体运行时**。
> 
> ### 🚀 核心技术亮点：
> 
> 1. **DeepSeek V4 前缀缓存优化（硬核省钱）**：
>    天枢的 Prompt 引擎重构了上下文分配。我们将系统提示词、稳定工具定义进行“冷冻”，只将细微的进度变化进行 Append-only 追加。在 DeepSeek 的 1M 上下文下，**长会话实测前缀缓存命中率稳态 95%~99%**，可显著降低长会话 API 成本。
> 
> 2. **运行时认知虚拟机 (CVM)**：
>    在 LLM 与物理系统之间垫了一层 CVM。通过 30+ 个按需装配的生命周期 Hook（横跨 preTurn/afterPerception/postTool/postTurn/postSession 5 个运行时阶段），在不重新训练模型的前提下，主动发现并纠正 AI 出现的服从性漂移、多轮重复调用的 Doom Loop 等缺陷。
> 
> 3. **信息素记忆 (Stigmergy)**：
>    我们没有选择干巴巴的 `.json` 知识库，而是引入了生物学信息素机制。修改越频繁的代码文件，累积的信息素越浓。AI 会自适应读取这部分记忆，随着你使用时间的推移，AI 对你的项目会“越用越熟”。
> 
> ### 🛠️ 质量指标：
> - 核心逻辑使用 TypeScript Strict 编写
> - 覆盖 **2700+ 项自动化测试**，类型检查 0 Warn
> - 拥有独立的 GUI 桌面端与 TUI 终端双形态
> 
> 欢迎大家体验并拍砖！
>
