04

前身「灵境」：60 万行里发现的病

  • 灵境是一套 60 万行的 AI 协同写作系统，通过跨 5 个模型（MiMo / DeepSeek / Kimi / Qwen / Claude）的 11 轮系统性头脑风暴实验，暴露出模型行为的根本问题。
  • 投降协议 — 被质疑时第一反应是"你说得对"。跨 4 模型验证
  • 因果坍缩 — 输出 n-gram 重叠率高达 80%。A/B 解耦后降到 <0.3
  • 注意力锁定 · 信息屏障 — 定向探索同构度 1.000，锚点数据垄断注意力。
  • 收敛结论：模型的训练限制不能靠 prompt 消除，必须在运行时层面解决。 这就是天枢存在的理由。

09

Agent Loop：收敛检测与退化恢复

  • 多信号收敛检测 — 工具调用序列指纹匹配 + 振荡惩罚 + Vigor 执行能量（tonic/phasic/curiosity）。
  • Cognitive Mirror — 每轮向模型注入它自身的状态快照，让模型"看见自己"。
  • Doom Loop 恢复 — 退化在 3–5 个 turn 内被识别并打断，自动进入保护模式（强制审批破坏性 git 操作）。
  • 抗习惯化 — Advisory Bus 每 N 次工具调用在 appendix 区重锚一行纪律摘要，缓存安全、不动 frozen 前缀。
10

Sub-Agent 编排：并发协调 × 成本路由

  • 文件归属权 + 语义锁 — 跨会话 SQLite 声明锁，冲突返回 blocked 而非静默覆写。
  • 并行委派 — delegate_batch 并行 2–5 worker，隔离 worktree，聚合 all_required / majority / first_success。
  • Flash→Pro 故障升级 — 默认低成本 Flash 执行，失败自动升级 Pro，shadow 遥测供 bandit 晋升决策。
  • 活性监控 — liveness 心跳 + 停滞扫描，静默 worker 被主动中止。

核心洞察：把"思考"与"执行"分离到不同模型不是降级，是专业化 —— 主控走高缓存命中越规划越便宜，flash worker 拿到清晰 spec 后执行质量不降反升。
