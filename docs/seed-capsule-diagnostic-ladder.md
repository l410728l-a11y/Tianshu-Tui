# 诊断手段阶梯 — 复杂问题排查方法论胶囊

> 复盘来源：session 20b9714e（凭接口签名推断 `checkPositive` 行为语义，三处误判）
> 与 virtue-verification-loop 交付复盘。核心教训：结构信心 ≠ 行为信心。
> static.ts 诊断循环挂骨架版，本胶囊为完整版，经 recall_capsule(诊断阶梯) 按需拉取。

<seed-capsule star="诊断阶梯" sealed="2026-07-11" gist="复杂问题排查的六级手段升级路线：取证→读实现→微探针→复现→基线→多视角">
排查手段阶梯——六级升级路线。从匹配问题量级的层级进入，不逐级履历；
上一级连续 2 次无新信息即升级到下一级，不原地重复。

① grep/read 取证（结构定位）
   回答"谁在哪里、谁调用谁、谁导入谁"。产出：文件/函数/调用关系的存在性。
   陷阱：这一级建立的是结构信心——接口存在、签名匹配、调用链通。它不能
   回答任何"运行时会发生什么"的问题。

② 读内部实现（行为语义）
   打开目标函数的函数体：switch 分支、正则、边界条件、默认值、空集合语义。
   结构信心来自接口验证，行为信心来自实现验证——两者是不同的操作。
   实例：checkPositive 的 tool_appears 谓词在 tools=[] 时语义是"任意工具
   即满足"——这写在 switch 实现里，不在接口签名里。凭签名推断 = 误判。
   适用判据：你即将依赖/接线某函数的行为，而不只是它的存在。
   bash 形态：grep -n/-c/-o、wc -l、head -n 等取证型只读命令与本级等价。

③ 微探针（15 秒实测）
   npx tsx -e "一行脚本" 或 .rivet/scratch/ 下的一次性测试文件，直接驱动
   目标函数、打印真实输出。读实现仍有歧义时（复杂状态机、隐式依赖），
   实测是唯一裁决。探针文件收尾时清理；scratch 路径不计 TDD gate。
   "测试太贵"是断言不是事实——先花 15 秒打探针再下结论。

④ 最小复现测试（RED 锁根因）
   遇悖论（两个已验证事实互相矛盾）或改动方案已成形时：写最小失败测试
   钉住预期行为，RED → 修 → GREEN。复现测试是最廉价的决定性证据——
   3 分钟的探针比 6 个文件的推理链更有说服力。

⑤ git 基线对照（回归专用，可直接跳入）
   "以前能跑现在不能"类问题不从 ① 爬起——git log 找可疑提交 →
   git bisect / checkpoint diff → 回归清单逐项定位。5 轮只读诊断空转
   （零写入）= 必须切到这级，不再读更多文件。

⑥ 多视角（结构性卡死时）
   同一视角连续无进展：explore 分身并行查 3+ 个独立方向；council 交叉
   审视；换星域 authority（如 yaoguang 验复现、tianquan 审架构）。
   到不相关领域找碎片，在碎片间寻找收敛（天璇法）。

三条横切纪律：
- 被拦不弃：工具被闸门拦截时读拦截文案里的出路 → 执行替代路径 →
  无替代转只读取证 + scratch 探针。被拦是策略约束不是失败证据，
  不进结论、不因此收窄排查范围。
- 焦虑对照：任何"上下文紧张/交给新会话"的判断必须先看 mirror ctx
  字段 / session_vitals 实测（≥70% 才算紧张）。10% 使用率时建议
  收尾是习惯性焦虑，不是审慎。
- 悖论即切换：静态阅读走入悖论 → 停止读更多文件，立即降到 ③/④ 级
  用物理实验裁决。同一函数读 3 个文件仍未排除矛盾 = 硬性触发器。
