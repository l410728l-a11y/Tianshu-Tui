# v2.19.0 迭代记录

> 日期：2026-07-12

## 功能变更

### 1. 跨端性能深度优化（Wave 0-5）

针对长会话首屏卡顿、高输出阻塞、跨会话搜索卡死等问题，进行全链路性能优化：

- **Wave 1 桌面重放与多会话调度**
  - 服务端 replay 增加 4ms 时间片 + `setImmediate` 让出事件循环，保留顺序与 gap backfill。
  - 桌面 event hub 改为每帧最多 400 事件 / 6ms 分片 drain，避免大同步 reducer 阻塞主线程。
  - 新增 session-open 性能预算，记录 first-fold / first-content / first-interactive。

- **Wave 2 CLI 监测、分级 flush 与稳定段缓存**
  - 新增 `TuiPerfMonitor`（仅 `--debug-perf` 或 `RIVET_DEBUG_TELEMETRY=1` 启用）。
  - write-batcher 增加 generation-safe `flushNow`，消除 stable commit 后重复 render。
  - stream-renderer 增加实例级 64 项 LRU，缓存 ≤16KB 稳定段。
  - 新增 shutdown-sequence 保证 perf-summary 落地。

- **Wave 3 工具 stdout 流控与桌面事件减量**
  - 新增 `OutputStreamBudget`：首 chunk 立即、40ms/2KB 合并、总 UI 预算、单次截断标记。
  - `bash` / `run_tests` 接入流控，保持 raw output / artifact / 模型输入不变。
  - 服务端对同一 tool id 的 streaming `tool_result` 做合并，减少桌面事件数。

- **Wave 4 跨会话搜索去阻塞**
  - 搜索抽出到 `session-search`，使用 `fs/promises` + 并发上限 4 + 批次间让出事件循环。
  - 桌面端接入 `AbortSignal` + 250ms debounce，避免旧响应覆盖。

- **Wave 5 prefix-cache 只观测与回归门禁**
  - 测量 `ttftMs`，聚合 `outputRawBytes` / `outputTrimmedBytes` / `outputFilterIds` / `toolUiEvents`。
  - telemetry-writer 放行 `perf-summary` / `vitals-lite`。

### 2. TUI 图片粘贴与识图桥

- TUI 端支持图片粘贴，自动转 base64 并透传给主控。
- 新增 `vision-service`，当主控模型不支持多模态时，由专用识图模型生成文字描述再进入对话。
- 桌面端设置增加「识图模型」配置卡片。

### 3. 桌面端版本更新说明

- 每次版本更新后首次启动桌面端，自动弹出「What's new」弹窗。
- 「设置 → 关于/帮助」可查看历史更新日志。
- 检查到新版本时，更新面板可直接查看对应版本说明。

### 4. 华盖星域

- 新增星域「华盖（huagai）」：长程守信、守昼托举、追 blocker、不让虚假完成偷走最后一英里。
