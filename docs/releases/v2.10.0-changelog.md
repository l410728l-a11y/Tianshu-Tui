## 功能变更

### 1. Review Discipline 内化（审查纪律运行时强制）

四条审查纪律从 CLAUDE.md 的文档级约束升级为 `deliver_task` 的结构化门禁：

| 纪律 | 实现 |
|------|------|
| 不可同上下文自我审批 | `reviewDepth` 数值穿 delegate 边界，子代理结构性跳过 ReviewRouter |
| 修复前 spawn adversarial_verifier | `routeReviewWorkflow` 按规模自动路由 L1/L2/L3 审查工作流 |
| 改 X 必跑既有测试 | ReviewRouter L2 档自动 spawn verifier 拿命令+输出证据 |
| "测试全过" fail-closed | `deliver_task` RED 门禁拦截无证据的绿声明 |

- **默认开启**（C=C1），可用 `RIVET_REVIEW_DISCIPLINE=0` 关闭
- 提交：`5110a28`（config + docs）、`5bcf7d3`（reviewDepth 17 文件）、`8cd2620`（死代码清理）

### 2. Intent Retrieval Router（意图检索路由）

用户消息进入后在主模型行动前先判断任务真实类型，注入检索方向提示，降低被关键词锚定的风险。

- **LLM 分类器默认开启**（`classifier: 'llm'` + `enabled: true`）
- Heuristic + LLM 双路径，LLM 失败自动 fallback
- 风险复验通过：风险2（无 onContentBlock 文本重复）+ 风险4（baseline must-source fallback）
- 提交：`8c6be6c`（默认开启）、`d3265ad`（风险复验）

### 3. Server 子系统上线

`/prompt` SSE 端点 + 双层鉴权 + H6 断连护栏 + go-live gate 全清。
