# v2.18.0 迭代记录

> 日期：2026-07-12

## 功能变更

### 1. 桌面端识图模型（Vision Model）配置

在「设置 → 集成」新增「识图模型」卡片。当主控模型不支持多模态时，可把用户粘贴/附带的图片先发给专用的识图模型（如 GLM-5.2、MiniMax-M3），再把文字描述送入主对话。

- 配置项：`agent.visionModel { provider, model, prompt?, maxTokens }`
- 仅显示已配置且声明 `supportsVision` 的模型。
- 保存后新会话生效。

### 2. TUI 图片粘贴支持

TUI 端支持通过终端 OSC 52 / 剪贴板粘贴图片，自动转 base64 并透传给主控。若主控模型不支持识图且已配置 vision bridge，则先由识图模型生成描述再进入 prompt。

### 3. 桌面端版本更新说明

新增版本发布说明系统：

- 每次版本更新后首次启动桌面端，自动弹出「What's new」弹窗。
- 「设置 → 关于/帮助」可查看历史更新日志。
- 检查到新版本时，更新面板可直接查看对应版本说明。

## 配置变更

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `agent.visionModel.provider` | string | 识图模型所属 provider |
| `agent.visionModel.model` | string | 识图模型 ID |
| `agent.visionModel.prompt` | string? | 自定义描述提示词 |
| `agent.visionModel.maxTokens` | number | 最大描述 tokens，默认 1024 |

## 维护说明

发版前请在 `docs/releases/vX.Y.Z.md` 写入本次更新说明，构建时会自动打包进桌面端。
