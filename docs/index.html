缓存为什么能做到 99.6%

不是魔法的 prompt 工程——是字节级前缀布局，精确控制每层的冻结与改写时机。

Frozen · 冻结层

System + Tools + Stable Context

会话开始时冻结，永不改写。每轮请求前缀完全命中——这部分占 token 的 ~60%，成本仅 1/50。

成本: ~1/50 × 基准
Working · 工作层

Conversation Messages

用户消息 + Assistant 回复 + 工具结果。追加模式（append-only），历史轮次自动命中前缀缓存。

新轮次: 基准成本
Appendix · 附加层

Dynamic Inject (~200B/turn)

进度信号、advisory、git status。放在消息末尾——前面的 frozen+working 层缓存不受影响。
