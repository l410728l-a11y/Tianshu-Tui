> 日期：2026-07-16 · 范围：Wave 0–5 全部落地 · 缓存类别：全波 K0/K1（无稳定前缀翻转）

## 背景

第一波收束解决了知识沉淀与召回管线。本波针对四条 harness 断链：
① 有损工具输出不可恢复；② CVM/advisory/SR 注入无出口计量；③ 压缩与 advisory
效能无账本；④ worker 委派信任链（验证→reward→未来派发）存在类型死端。

执行前用两个只读侦察固定了事实基线（改动前状态，作为前后对照存档）：

- **B 波计量出口**：CVM 开销只经 `PressureMonitor.recordCvmInjection` 计量
  projection/toolContext；AdvisoryBus appendix/SR、`injectUserMessage`、
  `appendSystemReminder` 全部漏计；无 source 分类枚举。
- **D 波 worker 验证链**：`RoutingEvent.verificationOutcome` 字段存在但全仓无
  producer；wave-gate 仅在 plan 波间生效；失败 worker 无「回原 worker 修复」
  闭环；真实 reward 走 team_wave 旁路，与 RoutingMetricsCollector 断线。

## Wave 4 — Worker 委派信任链

| 项 | 落点 | 内容 |
|----|------|------|
| D1 主控写产出验证门 | `src/agent/worker-write-gate.ts` + `hands-session.ts` | 复用 `evaluateWaveGate`（memoized scoped typecheck + 白名单命令），不新建 runner。条件矩阵：只读跳过；passed 放行；failed 回**原 worker 一次**有界修复（同 runAgent 闭包，结构上无换模通道）；blocked 环境中性回主控（不修复不惩罚）；`exit 0 + 0 passed` 判无效验证永不 passed；worker 声称 passed 但 gate failed → falseGreen。隔离 worktree 先 `provisionSnapshotDeps` 补依赖链接；diff 在 gate 后收集（含修复轮改动）。kill switch：`RIVET_WORKER_WRITE_GATE=0` |
| D2 verificationOutcome 事实流 | `turn-end.ts` / `model-routing-shadow.ts` / `model-routing-shadow-controller.ts` / `reward-loop.ts` / `loop-factory.ts` | producer 接通：turn-end 记录切换事件时取 EvidenceTracker 最新 verification；shadow event 新增同字段；`routingRewardInputFromEvent` 映射 `verificationPass`（blocked 保持 undefined 中性）。消费到 `buildHistoricalModelRewards` 为止，实际路由影响仍锁在 `efeRouting.enabled` 门后（默认 shadow only）——先 shadow 后晋级 |

## 验证结果

- `npm run typecheck` 干净（顺手修 W2 遗留测试一处 `budget` 可空性）。
- 全量 5451 测试：5449 过。唯一失败 `src/lsp/__tests__/integration/client-typecheck.test.ts`
  为满载机器 tsc 120s 超时，单独重跑通过——既有 CPU 争抢环境问题，与本波无关。
- 本地字节前缀门禁（cache-prefix-replay + engine-cache-stability + context-sr-append，62 用例）全绿。
- 真实 provider 烟测：环境 `DEEPSEEK_API_KEY` 返回 401（key 无效）。
  **服务端命中率本轮未验证**，不伪报；本地确定性门禁不受影响。有效 key 就绪后
  跑 `npm exec -- tsx scripts/verify-cache-hit-rate.ts` 补发布级验证。

## 设计偏差

- D1 矩阵中 blocked 的处理：计划写「回主控」，实现将 result 状态置为 `blocked`
  并附环境中性风险行——贴近 wave-gate「未验证 ≠ 验证通过」硬门禁语义，比只加
  注释更 fail-closed。非 TS 项目不受影响（无 .ts 改动时 typecheck 直接 clean）。
