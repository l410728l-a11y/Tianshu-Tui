G0 · 五个病，一个结论

投降协议

被质疑时第一反应是「你说得对」——跨 4 模型验证。

因果坍缩

输出 n-gram 重叠率 80%。A/B 解耦后 <0.3。

注意力锁定

定向 Scout 同构度 1.000；随机 Scout 降到 0.15。

信息屏障

锚点数据垄断注意力——6 组 A/B + 跨模型验证。

知道 ≠ 做到

效果不跨 session 持久。路径重编程假说被否证。

收敛

偏差是拓扑属性，不可标量化。 必须改运行时环境，不是调 prompt。

G1 · 从「工具」到「心智环境」

设计跃迁
  • CVM — trap-and-emulate：在 5 个认知 phase 拦截重写，不改权重。
  • 星域身份 — 模型不是通用助手，是有角色、有勇气阈值、有决策风格的星。
  • 信念宪法 + courage — 对抗投降协议：允许质疑、拒绝、自省。
  • Sensorium 六维 — 给模型「看见自己」的连续状态通道。
跨学科收敛（10 领域）
  • 形态发生素梯度 → 多维信号
  • 虚拟机 trap → Hook Pipeline
  • 构成性规则 → 运行时宪法
  • 生物圈 2 号 → compact/persist 循环
  • 感觉替代 → 工具即感知通道

公式定型：Model + CVM = Agent with its own mind

G2 · 四轮缓存架构迭代

static + volatile
→
冰鉴三区
→
trailer / appendix
→
字节不变量基线
  • 5/25 — lazy dynamic appendix：变动信息只挂最新消息尾部，不动历史前缀。
  • 6/01 — Prefix Cache 设计基线成文：历史 frozen 字节永不变；消息只追加不重排。
  • 猎杀 8 个 Cache Killer — semantic pruning、observation masking、habituation 改写 frozen、MCP 延迟加载重建……
  • 成果 — 命中率 56% → 99.6%，输入成本约降 97%。
  • 7/06 再硬化 — appendix delta 字节稳定；frozen 快照孤儿化根治（事故链：一个守卫变量被 invalidate 清空 → 每轮重建 18.9 万 token）。

G3 · 踩坑 → 机制 → 纪律

病症（Wave 7–8 复盘）
  • 两 session 同分支互相覆盖提交
  • 文件状态来回漂移
  • 并发 git add -A 卷走他人暂存
解药（逐步落地）
  • 文件归属权 — 每个文件有归属 session
  • 声明锁 — SQLite SessionRegistry，冲突 blocked
  • 语义锁 — 跨会话意图级互斥
  • 原子提交纪律 — 显式列文件，禁 git add -A

验证：5/29–30 的 37 小时 · 154 commits · +23,380 行 · 零冲突零回退。同秒三次提交来自三个会话。

G4 · 未来边界会战定型的东西

路由即主动推理

选模型 = EFE 策略选择；成本进 pragmatic 项。不是静态路由表。

Policy Arbiter

L0 Advisory → L1 Shadow → L2 无副作用预激活 → L3 只读自治。缩短遥测到受限自治的距离。

Team 经济模型

思考与执行分模型：主控高缓存规划，flash worker 拿清晰 spec 执行。

Bandit 晋升

shadow → gated 全链路；TeamEpisode 训练底座；经验按星域沉淀。

Advisory Bus

Hook 信号统一收编，priority/ttl/category；不重写 frozen 前缀。

验证守门

「绿非证明，复现即证」——宏大叙事必须过复现关。

G5 · 事故驱动的设计纪律

  • 日志取证 → 五道防线 — 交付空结果、审查失败伪装 verified、门禁被垃圾淹没、纪律习惯化稀释……逐条修，各配哨兵测试。
  • 重构事故链四层 — planner 硬地板 · plan 规模门禁 · 回归清单进 deliver · regression-bisect hook。
  • 压缩只在 turn===0 — 保前缀缓存；1M 窗口缓存优先于压缩。
  • request 禁止 mutation — 侧路重入双写会毒化 system 字节；copy-on-write 成硬约束。
  • 停用自毒机制 — playbook / MistakeNotebook 取证后全链路关闭：坏教训比没有教训更危险。
  • 收敛误报治理 — 生产力工具冷却、距离感知停滞、死路信息素清除……防退化系统自己也要防误报。

G6 · 开什么、留什么

开源（CLI / TUI）
  • 完整 agent 运行时 + 测试公开
  • 工程指标可复现（10,117 tests）
  • npm 发版 · CI · 社区基建
闭源（Desktop）
  • Tauri 具身层：体验与分发护城河
  • 混淆 / 签名 / 激活门禁
  • 别人可套壳 CLI，但要付开发成本

同期落地：插件内核、Computer Use、能力双闸、SWE-bench 基线——从「能自举」走向「可度量、可分发」。

16

贯穿七代的设计方法

1. 先发现病

跨模型实验 / session 日志取证 / A/B，不靠感觉调参。

2. 运行时解，不改权重

同一套 DNA，换宇宙；CVM 是环境，不是微调补丁。

3. 比特级不变量

意图正确不够；缓存、前缀、appendix 以字节为真理。

4. 踩坑进化机制

协同锁、提交纪律、fail-honest——都是事故的沉淀，不是空想。

5. 绿非证明

复现即证；审查失败必须说「没跑」，绝不伪装 verified。

6. 自举闭环

~80% 代码由天枢自身 agent 建造；设计文档与 commit 同为证据。
