# 设计审美判断力 — 高层指导到界面的转化方法论

> 场景来源：仅凭一句高层指导（"做一个设置页"）产出美观、符合人体工学且功能
> 完善的界面。与文曲域信条同源（克制、媒介诚实、结构美感自然结晶），
> 本胶囊给可操作展开。方法论 recall-only，冻结前缀只挂 gist 一行。

<seed-capsule star="设计审美" sealed="2026-07-11" gist="高层指导→界面的判断力框架：视觉词汇先行→四要素判断→人体工学→3变体→渲染自检">
设计审美判断力——UI/UX 类任务的执行骨架与品味基线。

① 视觉词汇先行（动手前）
   读现有设计系统：theme tokens、色彩语义键、间距刻度、既有组件的密度
   与圆角习惯。延续既有声音，不凭空发明新调色板；用语义色 / 和谐 oklch，
   永不裸写 hex。没有设计系统时，先立最小 tokens（一组间距刻度 + 两级
   层级色）再写组件。

② 判断力四要素（每屏自问）
   - 层级：一眼能否分出主次？主操作是否唯一且显著？
   - 间距：留白是否成刻度（4/8 栅格），还是随手 margin？
   - 对比：文本可读性（WCAG AA 起步）、状态差异是否可辨？
   - 节奏：重复元素是否对齐成韵律，还是各排各的？
   四要素不过关的"创意"是噪音；先过基线再谈风格。

③ 人体工学（功能完善的底线）
   - 触达热区：可点目标 ≥ 44px，高频操作靠近拇指/光标动线。
   - 焦点顺序：Tab 序 = 视觉序 = 逻辑序；键盘可完成主流程。
   - 可达性：不只靠颜色传达状态；图标配文字或 aria 标签。
   - 状态完备：空态 / 加载 / 错误 / 溢出（长文本、大数字）四态都有着落，
     不只设计 happy path。

④ 高层指导→界面的流程
   意图澄清（受众、场景、成功判据，一次问全）→ 3 变体草案（一版贴既有
   模式、一版调密度/层级、一版换交互结构；给取舍不给堆砌）→ 选定后实现
   （复用既有组件优先）→ 渲染自检（衔接交付纪律：browser/computer_use
   截图核对，或提请人工过目）。

横切信条：克制——删到不能再删才加；媒介诚实——网页像网页、终端像终端，
不做假拟物；缺真实素材时用诚实占位符，不做劣质模仿。
