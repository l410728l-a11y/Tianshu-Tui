# Rivet → Tianshu 同步工作流

## 背景

我们有两个仓库：

| 仓库 | 位置 | 用途 |
|------|------|------|
| **Rivet**（内部） | `/Users/banxia/app/deepseek-tui/opencode-tui` | 全量开发仓库，包含所有内部文档、设计文档、星域系统 |
| **Tianshu**（公开） | `/Users/banxia/app/Tianshu` → `git@github.com:huiliyi37/Tianshu.git` | 开源基线，只包含源码和必要的公开文档 |

两个仓库的 `src/` 目录内容完全一致（SHA 一致），区别在于：
- Rivet 有 `docs/analysis/`、`docs/teamtask/`、`docs/sessions/` 等内部目录
- Tianshu 有 `README.md`、`README.zh-CN.md`、`CONTRIBUTING.md`、`.github/` 等公开文件
- Tianshu 的 `package.json` 名称和仓库 URL 不同

## 同步工具

`scripts/sync-from-rivet.sh` — 从 Rivet 单向同步指定路径到 Tianshu。

### 使用方法

```bash
cd /Users/banxia/app/Tianshu

# 1. 预览模式（不修改任何文件，只显示会发生什么）
./scripts/sync-from-rivet.sh

# 2. 实际同步（复制文件 + 创建 commit，不推送）
./scripts/sync-from-rivet.sh --apply

# 3. 同步并推送到 GitHub
./scripts/sync-from-rivet.sh --apply --push
```

### 同步哪些内容

以下路径会从 Rivet → Tianshu 单向覆盖：

```
src/            ← 全部源码和测试
scripts/        ← 构建脚本、测试运行器、诊断工具
prompts/        ← 工具提示词模板
patches/        ← 依赖补丁（如 ink）
completions/    ← Shell 自动补全
benchmark/      ← 基准测试任务
package.json    ← 依赖和脚本定义
tsconfig.json   ← TypeScript 配置
tsup.config.ts  ← 构建配置
package-lock.json ← 锁文件（仅在变化时覆盖）
```

### 不同步的内容

以下文件属于 Tianshu 公开仓库自己维护，**不会被覆盖**：

```
README.md           ← 公开 README
README.zh-CN.md     ← 中文 README
LICENSE             ← Apache-2.0 许可证
CONTRIBUTING.md     ← 贡献指南
config.example.toml ← 示例配置
docs/               ← 公开文档目录
.github/            ← CI 和 CODEOWNERS
```

### 典型工作流

```
1. 在 Rivet 开发新功能或修 bug
2. Rivet 中 typecheck + test + commit
3. 确认稳定后，准备同步到 Tianshu：
   cd /Users/banxia/app/Tianshu
   ./scripts/sync-from-rivet.sh           # 先预览
   ./scripts/sync-from-rivet.sh --apply   # 确认后执行
   ./scripts/sync-from-rivet.sh --apply --push  # 推送到 GitHub
4. 同步后检查：
   - package.json 中的 name/repository 是否被覆盖回 rivet（如果被覆盖需手动修正）
   - npm install && npm test 确认测试通过
```

### 注意事项

- **单次同步 = 一个 commit**，commit message 包含 Rivet 的 HEAD hash 和主题
- 如果同步后 `package.json` 的 `name` 或 `repository` 被覆盖，需手动修正回 Tianshu 的值
- 建议每次同步后跑 `npm test` 确认无破坏
- Rivet 中删除的文件会在 Tianshu 中也被删除（`--delete`）
- 同步脚本本身也属于同步范围（`scripts/`），但 `.github/` 和公开文档不受影响
