# 架构审查 Checklist — 从 compact/cache 审查中反推的方法论缺口

> 来源：2026-06-23 compact 压缩系统架构审查全链路
> 审查对象：原始审查计划 → 三轮源码核实 → 两份调查报告 → 三个修复提交 → 分层归档方案

## 背景

一次完整的架构审查链路：外部审查报告提出 14 个问题 → 天枢逐条核实前 3 个 → 发现审查有盲区（遗漏 T7 请求时折叠缓解机制）→ 调查报告补充核实 → 三个提交修复 P0-P7 → 新方案提出范式升级（归档+召回）。

从这条链路中反推出五个方法论缺口——不是"做错了什么"，而是"每次推进都暴露了一个系统性盲区，但补得太晚"。

## 五个缺口

### 1. 缓解机制盲区
**现象**：第一轮审查只读了触发代码（compaction-controller.ts），没读缓解代码（engine.ts T7 请求时折叠），导致严重度高估。
**根因**：审查时只看"问题路径"不看"缓解路径"。同一架构里保护缓存的机制有 8 层，只读触发层会系统性高估严重度。
**对策**：架构审查第一步是"枚举所有缓解机制"而不是"读触发代码"。触发代码告诉你问题在哪，缓解代码告诉你问题有多严重。

### 2. 虚假绿灯检查
**现象**：P2.1 测试因 `makeController` 没传 `primaryClient`，`&& this.deps.primaryClient` 短路导致 60%/75% 分支从未被测到，但测试永远绿。
**根因**：短路条件让测试在不触发核心逻辑的情况下也能 pass。"声称覆盖了某路径"不等于"该路径被执行了"。
**对策**：审查测试时必须验证被测条件确实执行——检查 mock 对象的方法是否被调用（如 `streamCalled` 断言）。

### 3. compact 路径的 cache 影响时间线
**现象**：a1731b79 的 P3 把 persist 放在 llmCompact 之前，破坏了 prefix cache（persist 热更新 session memory → frozen base 重建 → llmCompact 请求 cache miss）。cd76b582 修正。
**根因**：在 compact 路径插入新操作时，没考虑该操作的副作用会波及 llmCompact 的 cache 复用窗口。
**对策**：compact 路径上插入任何新操作（persist、archive、inject、task-anchor）之前，画一条时间线——标注"llmCompact 复用 prefix cache 的窗口"，任何改变 frozen base 的操作不能在这个窗口内。

### 4. 范式反思步骤
**现象**：三个提交修了 P0-P7，全在"摘要丢弃"范式内打补丁。直到第四轮才有人问"为什么要丢弃？"——分层归档方案正是这个问题的答案。
**根因**：逐问题核实后缺少范式反思——"这些问题的共同根因是一个范式假设吗？"
**对策**：逐问题核实后加一个"范式反思"步骤——挑战共同根因假设。compact 的共同根因是"摘要有损且不可逆"，这个假设可以被"归档+召回"替代。

### 5. 同量多估算器
**现象**：maybeCompact 用 `session.getEstimatedTokens()`，T7 用 `estChars/4`（CJK 低估 3.3 倍），两套口径导致两个子系统压缩决策不协调。
**根因**：同一个物理量（token 数）被多个子系统各自估算，口径不一致导致决策错位。
**对策**：架构审查时 grep "同一物理量的所有估算器"——如果存在多个，口径必须一致。

## 审查 Checklist（可直接用于下次架构审查）

### 第一阶段：地图构建
- [ ] 枚举所有缓解机制（不只是触发机制）
- [ ] grep 同一物理量的所有估算器，确认口径一致
- [ ] 确认每个声称"覆盖了某路径"的测试是否真正执行该路径

### 第二阶段：逐问题核实
- [ ] 每个断言用源码交叉验证行号和逻辑
- [ ] 标注严重度时考虑缓解机制的实际减压效果
- [ ] 检查 compact 路径上新操作的 cache 影响时间线

### 第三阶段：范式反思
- [ ] 所有问题的共同根因是否指向同一个范式假设？
- [ ] 挑战这个假设会怎样？（是否有一个升级方案能让多个问题同时消失？）
- [ ] 当前补丁是在范式内修补，还是在为范式迁移铺路？

### 第四阶段：提交审查
- [ ] commit 粒度：一个逻辑单元一个提交
- [ ] 新增测试的反证能力：去掉实现后测试是否会红？
- [ ] persist/archive/inject 的副作用是否在 cache 复用窗口外？
